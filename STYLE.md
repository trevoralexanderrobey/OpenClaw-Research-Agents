# STYLE Guide (Phase 5 RLHF)

## Structured reasoning
- Use explicit sections with stable headings.
- Keep arguments concise, testable, and internally coherent.
- Preserve deterministic wording for fixed templates.

## Markdown formatting
- Use CommonMark-compatible headings and tables.
- Use fenced code blocks for code examples.
- Keep line endings as LF and avoid hidden formatting.

## LaTeX-compatible math
- Use inline math as `$...$` and display math as `$$...$$`.
- Avoid renderer-specific macros when a standard expression is available.

## Rubric table standards
- Include Criterion, Weight, and Guidance columns.
- Weights must be integer percentages summing to 100.
- Guidance text must be actionable and domain-relevant.

## AI-assistance disclosure
- Every draft must explicitly state AI assistance.
- Disclosure must be present in the document header.
- Do not remove disclosure text in later revisions.

## Human-review-required language
- Every draft must state human-review-required status.
- Final editorial judgment and platform submission remain manual.

## Policy prohibitions
- forbid concealment of AI assistance.
- forbid impersonation of humans.
- forbid detection evasion language.
- forbid masking synthetic origin.
- Do not add autonomous submission steps, credential handling, or login automation.
